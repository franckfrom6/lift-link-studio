====================================================================
PROMPT P3 — ENRICHISSEMENT DU CATALOGUE D'EXERCICES PAR DÉFAUT (6way)
Batch 2 : ~65 nouveaux exercices hardcodés
À coller dans Lovable — indépendant de P1/P2
====================================================================

Crée une nouvelle Supabase edge function `seed-default-exercises-batch2` (TypeScript/Deno).

Cette fonction ajoute de nouveaux exercices par défaut manquants dans `public.exercises`.
Elle est idempotente : elle ne reinsère jamais un exercice dont le nom existe déjà.

---

## Pattern de base

Même structure que `create-test-accounts/index.ts` :
- Client admin `SUPABASE_SERVICE_ROLE_KEY`
- corsHeaders `"*"`, gestion OPTIONS
- JSON body en entrée

---

## Interface d'appel

```ts
{
  dry_run: boolean;      // défaut true — si true, aucune insertion
  confirm?: string;      // "AJOUTER_EXERCICES" pour autoriser les insertions
  categories?: string[]; // filtre optionnel : n'insérer que ces catégories
                         // ex: ["pectoraux","dos"] — si absent, tout insérer
}
```

---

## Logique d'idempotence

Avant chaque insertion, vérifier que l'exercice n'existe pas déjà :
```sql
SELECT id FROM exercises
WHERE is_default = true
  AND (
    lower(trim(name)) = lower(trim($name)) OR
    (name_en IS NOT NULL AND lower(trim(name_en)) = lower(trim($name_en)))
  )
LIMIT 1;
```
Si un résultat existe, passer à l'exercice suivant (compter comme `skipped`).

## Comportement dry_run=true

Pour chaque exercice de la liste :
1. Vérifier s'il existe déjà (requête SELECT sur name + name_en, insensible à la casse)
2. Construire le rapport

Retourner :
```json
{
  "mode": "DRY_RUN — aucune insertion",
  "total_a_inserer": N,
  "total_dejà_presents": N,
  "a_inserer": [
    { "name": "...", "name_en": "...", "muscle_group": "...", "equipment": "...", "type": "..." }
  ],
  "deja_presents": [
    { "name": "...", "raison": "name ou name_en déjà en base" }
  ],
  "par_muscle_group": {
    "Pectoraux": 6, "Dos": 5, "Épaules": 5, "Biceps": 3, "Triceps": 3,
    "Jambes": 8, "Fessiers": 6, "Abdominaux": 8, "Cardio": 6, "Trapèzes": 2,
    "Full Body": 3, "Mollets": 2, "Étirements": 7
  },
  "next_step": "Relancer avec { dry_run: false, confirm: 'AJOUTER_EXERCICES' } pour insérer."
}
```

## Comportement dry_run=false

Guard :
```ts
if (!body.dry_run && body.confirm !== "AJOUTER_EXERCICES") {
  return new Response(JSON.stringify({ error: "confirm manquant ou invalide" }), { status: 400 });
}
```

Pour chaque exercice non déjà présent, insérer :
```sql
INSERT INTO exercises (
  name, name_en, muscle_group, secondary_muscle, equipment,
  type, tracking_type, public_cible, description,
  is_default, created_by
)
VALUES (
  $name, $name_en, $muscle_group, $secondary_muscle, $equipment,
  $type, $tracking_type, $public_cible, $description,
  true, null
);
```

**Filtre `categories`** (optionnel) : si `body.categories` est fourni, n'insérer que les exercices dont `muscle_group` est dans la liste, insensible à la casse.

**Idempotence :** vérifier l'existence avant chaque INSERT. Si déjà présent, passer sans erreur.

Retourner :
```json
{
  "applied": true,
  "inserted": N,
  "skipped_already_present": N,
  "inserted_ids": ["uuid1", "uuid2", ...],
  "timestamp": "ISO8601"
}
```
