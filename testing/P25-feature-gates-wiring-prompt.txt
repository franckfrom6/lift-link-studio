## P25 — Câblage des feature gates admin → UI

### Contexte
Le système de plan (`PlanProvider` → `useFeatureAccess` → `FeatureGate`) existe et
fonctionne correctement pour 4 features. Mais 26 features configurables dans l'admin
n'ont aucune gate UI : les désactiver ne change rien côté utilisateur.

Ce prompt câble les 7 plus impactantes + expose `ai_chat` dans l'admin.

---

### 1 — Ajouter `ai_chat` à l'UI admin (translations seulement)

Le code gate déjà `ai_chat` via `useFeatureAccess("ai_chat")` dans `useAIChat.ts` et
`AISidebarToggle.tsx`, mais la clé n'est pas dans le panneau admin — l'admin ne peut
pas la configurer.

**`src/i18n/locales/en/admin.json`** — ajouter dans l'objet `features` :
```json
"ai_chat": "AI Chat / VOLT"
```

**`src/i18n/locales/fr/admin.json`** — ajouter dans l'objet `features` :
```json
"ai_chat": "Chat IA / VOLT"
```

---

### 2 — `body_measurements` — StudentProgress.tsx

**Fichier :** `src/pages/student/StudentProgress.tsx`

Actuellement la section "Mensurations" est conditionnée par `isAdvanced` (préférence
d'affichage de l'utilisateur), pas par le plan. Remplacer la condition pour utiliser la
feature gate du plan.

**Import à ajouter** (si pas déjà présent) :
```tsx
import { useFeatureAccess } from "@/providers/PlanProvider";
```

**Dans le composant StudentProgress**, ajouter :
```tsx
const { isEnabled: canBodyMeasurements } = useFeatureAccess("body_measurements");
```

**Remplacer** le bloc `{/* Mensurations + photos — Pro only */}` (lignes ~220-241) :

```tsx
{/* Mensurations + photos */}
{isAdvanced && canBodyMeasurements ? (
  <>
    <ProgSection title="Mensurations">
      <BodyEvolutionSection />
    </ProgSection>

    <FeatureGate feature="progress_photos" showLocked>
      <ProgSection title="Photos">
        <ProgressPhotoGallery />
      </ProgSection>
    </FeatureGate>
  </>
) : isAdvanced && !canBodyMeasurements ? (
  <FeatureGate feature="body_measurements" showLocked>
    <ProgSection title="Mensurations">
      <BodyEvolutionSection />
    </ProgSection>
  </FeatureGate>
) : (
  <ProgSection title="Mensurations">
    <ProgEmptyState
      icon={ImageIcon}
      title="Disponible en mode Pro"
      description="Active le mode Pro pour suivre poids, masse grasse et photos de progression."
    />
  </ProgSection>
)}
```

### 7 — `FeatureGate` sans `showLocked` = rendu null (comportement attendu)

Pour les usages sans `showLocked` (StudentDetail bouton bilan, StudentWeek dropdown),
`FeatureGate` doit retourner `null` quand la feature est désactivée (comportement
actuel du composant — vérifier que c'est bien le cas dans `FeatureGate.tsx`).

---

### Comportement attendu après ces changements

| Feature désactivée dans admin | Effet visible |
|---|---|
| `ai_chat` | Bouton VOLT + chat disparu (déjà fonctionnel) |
| `body_measurements` | Section mensurations montre état verrouillé dans Progress |
| `strength_charts` | Graphique progression caché dans détail exercice |
| `ai_cycle_report` | Page bilan montre message bloqué ; bouton "Générer bilan" caché dans StudentDetail |
| `pdf_export` | Bouton télécharger PDF absent dans le bilan généré |
| `ai_week_optimizer` | Section adaptation IA cachée dans StudentDetail |
| `external_sessions` | Option "Loguer activité" absente du menu journée |

### Contraintes
- Ne pas casser la logique `isAdvanced` existante (c'est une préférence UI, pas un plan)
- Utiliser `<FeatureGate showLocked>` pour les sections importantes (montre un état
  "premium" au lieu de disparaître silencieusement)
- Utiliser `<FeatureGate>` sans `showLocked` pour les boutons/actions secondaires
  (ils disparaissent simplement)
- Aucun refetch supplémentaire — `useFeatureAccess` lit depuis le contexte déjà chargé
